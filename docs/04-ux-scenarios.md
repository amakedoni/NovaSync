# 04 — UX-сценарии

## Основной сценарий (happy path)

```
1. Пользователь работает в любом приложении
2. Нажимает Alt+Space
   → МГНОВЕННО (<100ms) открывается оверлей в центре экрана
3. Оверлей: тёмный полупрозрачный фон + панель по центру
   ┌─────────────────────────────────────────┐
   │  (фон: rgba(0,0,0,0.6) на весь экран)   │
   │                                         │
   │    ┌───────────────────────────────┐    │
   │    │  🔍 Type your question...     │    │
   │    │                               │    │
   │    │  [DeepSeek ▼]  [Explain      │    │
   │    │                 Translate    ▶│    │
   │    │                 Summarize     │    │
   │    └───────────────────────────────┘    │
   │                                         │
   │         Нажмите Esc чтобы закрыть       │
   └─────────────────────────────────────────┘
4. Пользователь вводит запрос, нажимает Enter
   → Оверлей исчезает (fade out 150ms)
5. Через ~0.5s в правом нижнем углу появляется виджет:
   ┌──────────────────────┐
   │ NovaSync          ✕  │
   │ ─────────────────── │
   │                      │
   │ (текст появляется     │
   │  по мере стриминга)   │
   │                      │
   │ ─────────────────── │
   │ [📋 Copy] [🔄 Retry] │
   └──────────────────────┘
6. Пользователь читает, копирует, закрывает
```

## Позиции и размеры

| Элемент | Позиция | Размер | Всегда поверх? |
|---------|---------|--------|---------------|
| Overlay | Центр экрана | 600×400 px | Да |
| Response Widget | Правый нижний угол (отступ 20px от краёв) | 420×520 px (resizable) | Да |
| Settings | Центр экрана | 500×400 px | Да |

## Поведение оверлея

| Триггер | Действие |
|---------|----------|
| Alt+Space (глобально) | Показать/скрыть оверлей |
| Enter (в оверлее) | Отправить запрос, скрыть оверлей |
| Escape | Скрыть оверлей |
| Клик вне панели | Скрыть оверлей |
| Открытие оверлея | Авто-фокус на строке ввода |
| Стрелка вверх (в пустом поле) | Показать историю запросов |
| Ctrl+V | Вставить из буфера обмена |

## Поведение виджета

| Триггер | Действие |
|---------|----------|
| Новый ответ | Появиться (slide-up + fade-in, 300ms) |
| Стриминг | Текст появляется токен за токеном |
| Кнопка Copy | Скопировать весь ответ в буфер, показать «Copied!» на 2s |
| Кнопка Retry | Повторить последний запрос |
| Кнопка ✕ | Закрыть виджет (fade-out, 200ms) |
| Drag (заголовок) | Переместить виджет |
| Resize (край) | Изменить размер |

## Дополнительные сценарии

### Быстрый вопрос (30 секунд)
```
Alt+Space → "сколько времени в Токио?" → Enter → смотрит виджет → закрывает
```

### Работа с экраном (V0.2)
```
Alt+Space → клик 📸 → скриншот → оверлей с превью →
"что это за ошибка?" → виджет с объяснением
```

### Контекстный вопрос (V0.2)
```
Выделил текст в браузере → Alt+Space →
(буфер содержит выделенный текст) →
Quick Action «Explain» → виджет с объяснением
```

## Принципы UX

1. **Скорость > анимация.** Оверлей <100ms, первый токен <3s.
2. **Разделение ввода и ответа.** Оверлей для вопроса, виджет для ответа.
3. **Не steal focus.** Оверлей получает фокус только для ввода текста.
4. **Always-on-top, но не мешает.** Виджет в углу, можно работать с другими окнами.
5. **Минимум кликов.** Один хоткей → один ввод → ответ виден.
6. **Тёмная тема всегда.** Меньше отвлекает, лучше читается код.
